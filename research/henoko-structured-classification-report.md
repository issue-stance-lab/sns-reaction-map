# 辺野古高校生死亡事故 SNS反応 再解析レポート

作成日: 2026-06-20

## 目的

Yahooリアルタイム検索から収集した辺野古高校生死亡事故関連の反応 311件について、従来の単純なカテゴリ分類ではなく、次の軸を分けて再分類した。

- `target`: 投稿が評価・批判している対象
- `stance_to_target`: その対象への態度
- `issue`: 主な論点
- `quote_direction`: 引用元に賛成しているのか、反対しているのか
- `review_required`: 記事化前に人間確認が必要か

この再解析は、反応マップや記事に直接流すためというより、分類精度を上げるための確認レイヤーとして使う。

## 分類精度の改善点

今回の再解析で改善した点:

- 「文科省判断に反発している意見」と「文科省判断を支持している意見」を分離できた。
- 「教育基本法違反を問題視する意見」と「教育基本法違反認定を問題視する意見」を分けられるようになった。
- 記事見出しを引用しているだけの投稿を、投稿者本人の立場として扱わない方針を入れた。
- 「何が萎縮させる危険やねん」「むしろ萎縮しろ」のような投稿は、文科省判断への反発ではなく、文科省判断支持側として分類できた。

まだ弱い点:

- `その他・分類保留` が65件あり、公開UIにそのまま使うには多い。
- `target` が `不明` の投稿が113件あり、リンク共有・見出し引用・短文投稿の扱いがまだ粗い。
- `review_required` が259件と多く、すべてを人間確認する運用には重い。
- `stance_to_target` に `批判` と `評価しない` が混在しており、UI用には再正規化した方がよい。

## UI・記事への使い方

現時点では、この構造化分類を既存の反応マップにそのまま上書きするのは避ける。

おすすめの使い方:

1. 既存UIは従来の編集済み分類を維持する。
2. 構造化分類は裏側の検証レイヤーとして使う。
3. 記事に載せる代表投稿だけ、`target` / `issue` / `quote_direction` / `review_required` を見て確認する。
4. UIには「評価する VS 評価しない」の大分類だけを出し、詳細分類は有料記事・分析ページ側で使う。

## 次の改善案

- `stance_to_target` をUI用に `評価する` / `評価しない` / `問題視` / `中立・保留` の4種へ正規化する。
- `review_required` は全件表示ではなく、代表投稿候補の抽出にだけ使う。
- `その他・分類保留` の65件を再分類対象として別ファイルに切り出す。
- DBの `classifications` テーブルに `target`, `stance_to_target`, `issue`, `quote_direction`, `review_required`, `review_reason` を追加する。
- 既存ページとは別に、比較用ページ `henoko-structured-reaction-map.html` を作る。
