# Handoff: SNS反応まっぷ リデザイン

## Overview
「SNS反応まっぷ」(意見が割れる社会的テーマをSNS投稿のAI分類と投票で可視化するメディア)の既存サイト(https://issue-stance-lab.github.io/sns-reaction-map/)をリデザインしたもの。トップページ、個別テーマページ、X(Twitter)用ヘッダーバナーの3点。

## About the Design Files
このバンドル内のHTMLファイルは**デザインリファレンス**です。実装用のコードとしてそのまま流用せず、対象コードベースの既存環境(React/Vue/静的サイトジェネレータ等、または新規構築ならプロジェクトに最適なフレームワーク)でこの見た目・挙動を再現してください。

## Fidelity
**Hi-fi**: 色・フォント・余白・インタラクションは最終決定値として扱ってよい。ピクセルパーフェクトな再現を目指してください。

## Screens / Views

### 1. トップページ (Top.dc.html)
- **Purpose**: サイト全体のハブ。分類比率のサマリー、注目テーマ、割れ度ランキング、全10テーマの一覧を提供し、投票への導線を作る。
- **Layout**: 全セクション `max-width: 1120px` 中央寄せ、`padding: 24px`(`--space-6`)。セクション間は高さ1pxの罫線(`--color-divider`)。
  - ヘッダー: `.nav` — flexbox一列、左にロゴ(`.nav-brand`)、中央にテキストリンク、右にX(旧Twitter)リンク+CTAボタン。`position: sticky; top: 0`。
  - ヒーロー: kickerタグ→大見出し(3行、`font-size: 56px`)→リード文→CTAボタン+スタンス凡例タグ列。
  - 分類比率: 高さ64pxの横積み上げバー(4色、幅=%) + 下に4列グリッドの数値内訳(各列左に1px罫線区切り)。
  - 統計: 3列グリッド(分析済み投稿数/公開テーマ数/次回更新日)、各列左罫線区切り、下に最終更新メモ。
  - 注目テーマ: サーフェス背景(`--color-surface`)の1枚カード、左にタグ+タイトル+概要+ミニ積み上げバー+メタ情報、右に「詳細を見る」リンク(ホバーで矢印が右上に移動)。
  - ランキング表: `<table class="table">` — #, テーマ名+カテゴリ, 分類比率のミニバー(角丸pill), 割れ度スコア の4列。
  - 話題のテーマ: 見出し+カテゴリ絞り込みセグメントコントロール(すべて/政治・法律/生活・教育、クリックでグリッドをフィルタ)。カードは `auto-fill minmax(260px,1fr)` グリッド、gap 2pxを罫線色背景で表現(擬似ボーダー)。各カードはkicker+NEW/投票受付中タグ+タイトル+ミニ積み上げバー+投稿数メタ。
  - データの見方: 左にアクセントカラー太罫線のノート。
  - フッター: サーフェス背景、3列グリッド(概要文/テーマリンク/サイト情報リンク)+罫線+著作権表記。

### 2. 個別テーマページ (Topic.dc.html) — 「副首都法案・副首都構想」
- **Purpose**: 1テーマを深掘りし、投票を前面に、6つの論点を提示。
- **Layout**: `max-width: 1000px` 中央寄せ。ヘッダー/フッターはトップページと共通。
  - パンくず → タグ(カテゴリ+投票受付中) → H1 → リード文 → メタ行(分析対象件数/論点数/更新日)。
  - 現在の意見の割合: 高さ56pxの積み上げバー+4列内訳(賛成・推進/反対/慎重/どちらでもない)。
  - **投票ブロック**(最重要要素): サーフェス背景のカード。未投票時は2×2グリッドの大きな投票ボタン(各選択肢、クリックで投票)。投票後は結果バー(ライブ集計を模したパーセンテージ再計算)+回答内容を表示。
  - 論点アリーナ: `<details>`native要素を使ったアコーディオン6件。summary行に番号+タイトル+ミニ積み上げバー+シェブロンアイコン、開くと説明文。
  - データの見方ノート、フッターはトップと共通。

### 3. Xヘッダーバナー (X Banner.dc.html) — 1500×500px
- **Purpose**: X(Twitter)プロフィール用ヘッダー画像。ポップでデータ感のあるビジュアル。
- **Layout**: 固定サイズ1500×500、`overflow:hidden`。
  - 背景: 薄いドットグリッド(`radial-gradient` 26px間隔)。
  - 四隅に円形の「波」(2重の円、外側は濃色・内側やや薄い色を重ねて配置、コンテナ外にはみ出させて角だけ見せる): 左上=青、右上=赤、左下=緑、右下=オレンジ。
  - 紙吹雪: 円・ひし形(45度回転の正方形)・三角形(border trickの小要素)を12個程度ランダム配置。
  - 統計吹き出し: 白背景・角丸(pill/20px)・box-shadowのチップ、アイコン+太字カラー数値。上部5個、下部5個。
  - 左: 円グラフ(`conic-gradient`、42%青/20%赤/20%緑/18%オレンジ相当、白フチ`box-shadow: 0 0 0 6px #fff`)。
  - 中央: 巨大タイトル「SNS反応まっぷ」(Inter 900, 96px, ネイビー#14213d) + サブコピー。
  - 右: 4本のランキングバー(色付きドット+角丸プログレスバー、幅の違いで順位を表現)。

## Interactions & Behavior
- カテゴリ絞り込み(トップページ): セグメントコントロールのradio切り替えで、テーマグリッドを `group` フィールドでフィルタ表示(即時、アニメーションなし)。
- 投票(テーマページ): ボタンクリックで選択肢を確定 → その選択肢の分類比率が微増するようリアルタイム再計算 → 投票UIを結果表示に切り替え(このセッション内のみ、永続化なし。実装時はAPI送信+サーバー集計に置き換える想定)。
- 論点アリーナ: ネイティブ`<details>/<summary>`の開閉(JS不要)。
- 「詳細を見る」リンク: ホバーで矢印アイコンが右上に3pxトランスレート。
- リンクホバー: 全体的に `color: var(--color-accent)` に変化。

## State Management
- `category`(トップページ): 選択中のカテゴリフィルタ("すべて" | "政治・法律" | "生活・教育")。
- `voted`(テーマページ): null(未投票) | 選択した論点キー。投票後は分類比率を再計算して表示。
- サーバー実装時は投票をAPIに送信し、実際のSNS分類比率と合わせて集計・永続化する設計に置き換える。

## Design Tokens

### Colors
| Role | Hex | 用途 |
|---|---|---|
| Background | `#ffffff` | ページ背景 |
| Surface | `#f4f6fb` | カード・強調セクション背景 |
| Text | `#222222` (見出しは `#14213d`) | 本文/見出し |
| Accent (blue) | `#2f6fed` | 賛成、プライマリボタン、リンク |
| Accent-2 (red) | `#ef3b3b` | 反対 |
| Warn (orange) | `#f5a623` | 条件付き/慎重 |
| Positive (green) | `#22a559` | 保留/どちらでもない |
| Divider | `#dde1ea` | 罫線(1px) |

濃淡が必要な箇所は各アクセントの100〜900の12段階トーナルランプを使用(例: `accent-700 #2456bd` をリンクのホバー濃色に使用)。

### Typography
- Heading/Body: **Inter**(400/600/700/800/900)
- H1: 40〜56px / H2: 24〜26px / H3: 20px / body: 15px, line-height 1.6
- 見出しは `letter-spacing: -0.015em`

### Spacing
4 / 8 / 12 / 16 / 24 / 32 px (`--space-1`〜`--space-8`)

### Radius
- ボタン/カード: 3px
- タグ/セグメントコントロール/プログレスバー: 999px(pill)

### Shadow
- sm: `0 1px 2px rgba(20,20,20,.06)`
- md: `0 2px 8px rgba(20,20,20,.08)`
- lg: `0 8px 24px rgba(20,20,20,.12)`

## Assets
外部画像なし。アイコンはすべてインラインSVG(自作、外部CDN依存なし)。フォントはGoogle Fonts経由のInterのみ。
