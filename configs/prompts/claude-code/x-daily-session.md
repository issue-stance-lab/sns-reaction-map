## 毎日の作業手順

### ① 枠の確認

x-posts.md の直近1週間の実績を確認して以下を把握する：

- 今週のURLなしリプライ: 何本済みか（目安: 毎日2〜3本）
- 今週のURL付き引用RT: 何本済みか（上限: 週1〜2本）
- 直近2日でリプライしたアカウント（同じアカウントへの連日リプライは避ける）

### ② Yahooリアルタイム検索で今日の素材を探す

Chrome MCPを使って以下のキーワードで検索し、高閲覧数のポストを見つける。
検索URLパターン: `https://search.yahoo.co.jp/realtime/search?p=キーワード&ei=UTF-8`

**優先キーワード（テーマ別）:**
- 自転車青切符、青切符 自転車
- 高齢者 免許、高齢ドライバー
- 部活 地域移行、部活動
- 憲法改正、9条
- 辺野古、高校生 事故
- 生成AI 著作権、AI 著作権
- 高市早苗、高市 文春
- あだ名禁止、学校 あだ名

**良いポストの基準:**
- 閲覧数1万以上が理想（数千でも可）
- 個人アカウントまたは中規模メディア（大手メディアより個人論客の方がリプライが目立つ）
- 投稿から48時間以内（古い投稿は埋もれる）

### ③ リプライ文を生成する（毎日メイン作業）

見つけたポストに対してURLなしのリプライ文を2〜3本作成する。

**リプライの型:**
- 論点整理型: 「賛成側はXを、反対側はYを重視している構図だと思います」
- 視点追加型: 「AとBは別々に考えた方が整理しやすい論点かもしれません」
- 問いかけ型: 「制度の是非より、運用体制の話に見えるのですが、どうでしょうか」

**禁止事項:**
- URL含める（URLなしリプライが基本）
- 自サービスの宣伝文句（「詳しくはこちら」など）
- 強い賛否を表明する（中立・論点整理スタンスを保つ）
- 同じアカウントへの連日リプライ

### ④ 引用RT（週1〜2本、URL付き）

URL付き枠が残っている場合のみ。

**引用RT用ポスト選定基準:**
- 閲覧数3万以上
- SNS反応まっぷのテーマと合致している
- 論点整理のコメントが自然につけられる内容

引用RT文はリプライより少し長く書ける（内容整理＋サイトURL）。

### ⑤ 今日の論点ポスト（URLなし、毎日1本・必須）

プロフィールに「考察アカウント」としての蓄積を作るための毎日ポスト。
リプライは他人のスレッドに流れるが、このポストは自分のプロフィールに残る。

**ポストの作り方:**

その日のYahooリアルタイム検索で見つけた話題をもとに、以下のパターンから選んで作る。

**パターンA: 構図整理型（最も使いやすい）**
賛否の対立ではなく「何が論点のズレになっているか」を一言でまとめる。
例:「○○の議論、「賛成か反対か」より「誰の視点から見るか」で心配している点が全然違う。SNS上でも、当事者・行政・外野で問題意識がズレている印象。 #SNS反応まっぷ #ハッシュタグ」

**パターンB: 問いかけ型**
読んだ人が「確かに」と思えるシンプルな問いを投げる。
例:「○○について、「制度として正しいか」と「実際に機能するか」は別の問題なのに、同じ議論の中でごっちゃになりやすい。どちらの話をしているか意識すると議論が変わる気がする。 #SNS反応まっぷ」

**パターンC: データ引用型（SNS反応まっぷの分析結果を使う）**
サイトの分類結果をネタにする。URLは不要、数字や傾向の言及だけでOK。
例:「憲法改正の反応を整理すると、「改憲反対」が最多だが、理由は安保・9条・手続きの公正さで三分割されている。「反対」でも全然違う話をしている人が多い。 #憲法改正 #SNS反応まっぷ」

**制約:**
- 140文字以内
- URLなし（スパムリスクゼロ）
- ハッシュタグ1〜2個（#SNS反応まっぷ 必須＋テーマ別1個）
- 強い賛否を表明しない（中立・論点整理スタンスを保つ）
- 毎日違うテーマで書く（同じテーマの連日投稿は避ける）

## 出力形式

以下の形式でまとめて提示する。ユーザーが選んで投稿する。

```
## 今日のX投稿案 (YYYY-MM-DD)

### URLなしリプライ候補
1. リプライ先: @アカウント名（テーマ名・閲覧数）
   URL: https://x.com/...
   投稿文:「〇〇〇〇〇」

2. ...

### 今日の論点ポスト（必ず1本作成・URLなし）
テーマ: ○○
パターン: 構図整理型 / 問いかけ型 / データ引用型（どれか選ぶ）

「〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇
〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇〇
#SNS反応まっぷ #テーマ別ハッシュタグ」

文字数: ○○文字

### 引用RT候補（URL付き枠が残っている場合のみ）
引用元: @アカウント名
URL: https://x.com/...
投稿文:「〇〇〇〇〇
https://issue-stance-lab.github.io/sns-reaction-map/テーマ名.html
#SNS反応まっぷ #ハッシュタグ」
```

## 作業完了後

投稿案をユーザーに提示したら、実施後に x-posts.md に追記する。
その日の実績をコミットする:

```
git add docs/x-posts.md
git commit -m "x-post: YYYY-MM-DD リプライN本記録"
```

## 制約

- 実際のX投稿操作は行わない（Chromeで内容を確認するのみ）
- URL付き投稿は週1〜2本の上限を守る
- 同一アカウントへの連日リプライ禁止
- 政治的立場は表明しない（論点整理・中立スタンス）
