# SNS反応まっぷ デザインシステム

現行サイト（https://issue-stance-lab.github.io/sns-reaction-map/）から抽出。
新規テーマ作成、バナー画像、インフォグラフィック、マンガ、OGP画像のプロンプトに使用する。

## 1. カラーパレット

### ブランドカラー（スタンス4色）

| 名前 | Hex | CSS変数 | 用途 |
|------|-----|---------|------|
| Blue（賛成） | `#075ef2` | `--blue` | 賛成・推進。プライマリアクセント、リンク、CTAボタン |
| Orange（反対） | `#ff5b2e` | `--orange` | 反対。スタンスマップ、分類バー |
| Green（条件付き） | `#36b8a3` | `--green` | 条件付き・慎重。更新パルスにも使用 |
| Purple（保留） | `#9358e8` | `--purple` | 保留・どちらでもない |

> この4色は全ページ・全画像で統一。投票カード、スタンスマップ、分類バー、マンガのアクセント、インフォグラフィックすべてに適用。

### UI基盤色

| 名前 | Hex | CSS変数 | 用途 |
|------|-----|---------|------|
| Navy | `#071a3d` | `--navy` | 見出し、本文のメインカラー |
| Muted | `#637089` | `--muted` | 補助テキスト、キャプション |
| Background | `#f6f9fe` | `--bg` | ページ背景（やや青みのオフホワイト） |
| Panel | `#ffffff` | `--panel` | カード・パネル背景 |
| Line | `#e4eaf3` | `--line` | ボーダー、罫線 |
| Accent-soft | `#edf4ff` | `--accent-soft` | アクセント薄色（badge背景等） |
| Soft | `#f5f8fc` | `--soft` | サーフェス背景 |

### テーマ別アクセント（各テーマページのヒーローグラデーション）

テーマごとにヒーローの `--grad-1` / `--grad-2` を設定する。例:
- 副首都: `#4338ca` → `#312e81`（深い紫）
- AI著作権: `#075ef2` → `#287dff`（ブルー、topic-modern.css）
- 自由に設定可能だが、上記4スタンス色とかぶらないニュアンスを推奨

### シャドウ

| 名前 | 値 | 用途 |
|------|---|------|
| Shadow | `0 12px 34px rgba(18,53,99,.09)` | カード、パネル |
| Shadow-sm | `0 5px 18px rgba(18,53,99,.07)` | 小カード、stat |

## 3. レイアウト

### 幅とスペーシング

- **最大幅**: `min(1180px, calc(100% - 40px))`（トップ）/ `min(1180px, 100%)`（テーマ内パネル）
- **ヘッダー高さ**: 72px–76px（デスクトップ）/ 62px（モバイル）
- **セクション間隔**: `margin-top: 14px`（パネル間）/ `34px`（セクション間）
- **パネル内padding**: `30px 34px`（デスクトップ）/ `23px 17px`（モバイル）

## 4. コンポーネント仕様

### ヘッダー（sticky）

```
背景: rgba(255,255,255,.92) + backdrop-filter: blur(16px)
下線: 1px solid rgba(222,230,241,.9)
```

- 左: ロゴマーク（6つの色付きドット + 2本の楕円リング）+ テキスト
- 中: ナビリンク（`12px, weight 800`）
- 右: Xアイコン + アウトラインボタン + プライマリボタン

### ロゴマーク

6つの小円（11px）を時計回りに配置、2本の楕円リング（`±35deg`回転）で囲む。
色: Blue, Blue2(`#2786ff`), Orange, Green, Purple, Orange の6ドット。
白い3pxの`box-shadow`でドット間を区切る。

### ヒーロー（トップページ）

- 3列グリッド: コピー / スキャッタープロットCanvas / サイドパネル
- 背景: 白 + 薄いグリッド線（`rgba(20,99,255,.035)` 48px間隔）
- CTA: pill型ボタン（`border-radius: 999px`）、ブルーグラデーション + 大きなシャドウ
- Canvas: 4色のクラスターポイント + 接続線

### ヒーロー（テーマページ）

- ヒーロー画像を右半分にフェードマスク（`mask-image: linear-gradient(90deg, transparent 0, #000 26%)`）
- 左にコピー: パンくず → badge → H1 → question-line → lead → 30秒サマリー（3列グリッドカード）
- topic-modern.css 適用時: 白背景パネル + `border-radius: 20px`

### スタンスバー（分類比率）

```css
display: flex; height: 21px; overflow: hidden; border-radius: 6px;
/* 各色のspan。% に応じた width。白文字で%表示 */
```

- 高さはコンテキストで変化: 21px（カード内）/ 5px（ランキング）/ 4px（トピックリスト）
- 色順: Blue → Orange → Green → Purple

### 投票カード（vote-image-card）

```
aspect-ratio: 1;
背景: 論点イメージ画像 (background-size: cover)
オーバーレイ: linear-gradient(180deg, rgba(30,27,75,.25), rgba(30,27,75,.72))
ラベル: 白文字, weight 900, 16px, text-shadow付き, 左下配置
ホバー: translateY(-4px) + シャドウ拡大
border-radius: 8px (topic-modern時: 13px)
```

### マンガカード（manga-page-card）

```
border-radius: 8px (topic-modern時: 13px)
画像: aspect-ratio 2/3, object-fit: cover
下にキャプション: 13px, weight 900
ホバー: translateY(-4px) + シャドウ拡大
クリック: モーダルで拡大表示
```

### 2Dスタンスマップ（Arena）

- ダーク背景: `#0f0e2e` + arena-bg.webp
- Canvas: 白背景、角丸12px、ドロップシャドウ大
- フィルターボタン: pill型、アクティブ時アクセント色
- ヒートマップ + スキャッタープロット切り替え
- topic-modern.css 適用時: 明るいUI（白背景、薄ボーダー）

### ランキングカード

```
背景: 白, border: 1px solid var(--line), border-radius: 15px
ランク番号: 26px円, 1位=金(#f3b51d), 2位=銀(#9ea9b7), 3位=銅(#c47537), 4位以降=グレー
割れ度メーター: 5px高さ、背景#edf1f5、Blueグラデーションfill
スコア: 大きな数値 (18px, weight 900)
```

### フッター

```
背景: #07172f（深いネイビー）
テキスト: rgba(255,255,255,.72)
リンク: rgba(255,255,255,.62)
下部ボーダー: 1px solid rgba(255,255,255,.1)
3列グリッド: ブランド説明 / テーマリンク / サイト情報
```

## 5. 画像アセット仕様

### ファイル命名規則

```
{slug}-hero.webp            # ヒーロー画像 (テーマページ背景)
{slug}-vote-{N}-{key}.webp  # 投票カード画像 (4枚, 正方形)
{slug}-manga-page-{N}.webp  # マンガページ (3–7枚, 2:3比率)
{slug}-manga-charsheet-{name}.webp  # キャラ設定シート
{slug}-infographic-{key}.webp       # インフォグラフィック
ogp/{slug}.png              # OGP画像 (1200×630推奨)
```

### ヒーロー画像

- 用途: テーマページのヒーロー背景
- サイズ: 幅1200px以上、横長
- テイスト: テーマに関連する象徴的なシーン。半透明（opacity 0.18）でグラデーション越しに表示されるため、コントラストが強い構図が望ましい

### 投票カード画像

- 4枚1セット（各スタンスに対応）
- aspect-ratio: 1:1（正方形）
- 暗いグラデーションオーバーレイの上に白文字ラベルが乗る前提
- テイスト: テーマの論点を象徴するイラスト or 写真

### マンガ

- 3–7ページ（テーマの論点数に応じて）
- aspect-ratio: 2:3（縦長）
- キャラ設定シートで外見・性格を固定し、全ページでキャラ一貫性を維持
- フキダシの日本語テキストは画像内に含める（AI画像生成で対応可能）

### インフォグラフィック

- 論点ごとの視覚的な解説図
- 4色パレットに沿った配色
- テキスト込みの完成画像

### OGP画像

- サイズ: 1200×630px
- デザイン: テーマ名 + ブランドロゴ + スタンスバーのビジュアル

## 6. インタラクション

| 要素 | 効果 |
|------|------|
| カード全般 | `hover: translateY(-3px) + shadow拡大` |
| 投票カード | `hover: translateY(-4px) + shadow拡大`、クリックで投票確定 |
| マンガカード | クリックでモーダル拡大表示 |
| ナビリンク | `hover: color → var(--blue)` |
| CTA pill | `hover: ブルー濃色 + shadow拡大` |
| Reveal | IntersectionObserver で `opacity 0→1` + `translateY(14px→0)`、`prefers-reduced-motion` 対応 |
| スタンスマップ | Canvas上のポイントにホバーでツールチップ表示 |

## 7. 画像生成プロンプト共通指示

バナー・マンガ・インフォグラフィック・投票カードを外部AI画像生成ツール（GPTimage2等）で作る際の統一指示:

### 色指定

```
必ず以下のブランドカラーを使用:
- メインブルー: #075ef2（賛成）
- オレンジ: #ff5b2e（反対）
- グリーン: #36b8a3（条件付き）
- パープル: #9358e8（保留）
- ネイビー: #071a3d（テキスト・見出し）
- 背景: #f6f9fe（薄い青みのオフホワイト）
```

### フォント・テキスト

```
- 日本語テキストはNoto Sans JP相当のゴシック体
- 見出し: 太字（Black/900相当）、letter-spacing詰め
- 本文: Regular 400
```

### トーン&マナー

```
- クリーン・モダン・データ密度が高い印象
- フラットデザイン寄り（過度なグラデーション・光沢は避ける）
- 角丸は控えめ（8px–18px）
- 白背景 + 薄い青みのカード影
- 情報の階層を色と太さで表現（華美な装飾より構造の明確さ）
```

### マンガ特有

```
- キャラ設定シートを先に生成し、全コマでキャラの外見を統一
- フキダシ内の日本語テキストは画像に直接含める
- 背景は簡潔に（テーマの象徴的な場所・状況）
- 4色パレットをアクセントに使用
```
